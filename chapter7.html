🎯 考试通关实战手册

这不是知识学习，是考试武器库——只讲"考官怎么问、你怎么答、怎么赢"。

01 考试形式拆解

考试到底怎么考？

根据言秋老师（两年前考过）的经验："根据客户业务场景介绍功能，然后考官出题答辩"。

这不是闭卷笔试，是现场答辩通关。考官扮演客户或直接出题，你要现场介绍功能+回答追问。

考试三环节（推测）

环节形式你要做什么
① 功能介绍考官给客户场景，你现场演示/介绍企微管家怎么解决像给客户做产品演示，突出管家优势
② 追问答辩考官就你介绍的功能追问差异、原理、边界准确回答，不慌不乱
③ 实操/画图可能让你口述配置步骤或画CRM流程图能说出关键步骤

核心心法：你不是在"答题"，你是在"做一场产品演示+客户答疑"。考官想看到的是：你能不能像客户成功/销售一样，把产品讲清楚、把客户问题接住。

02 8模块演示话术（背下来直接用）

考官大概率从8个功能模块里挑场景让你介绍。下面每个模块给你一套可直接背诵的演示话术——客户场景+功能介绍+管家优势，三段式。

模块1：对外联系（联系我/加入群聊）

场景："某零售门店想让到店客户扫码加导购微信，怎么做？"

话术：用企业微信的「联系我」功能。创建一个多人二维码，最多支持100名成员，客户扫码后自动分配给导购。如果想要更精细的运营——比如按排班分配、设客户上限自动转接、区分新老客、分时段欢迎语——那就用企微管家的「活码」，它是联系我的7维超级升级版。

管家优势点：活码多了多人路由+分时段+时刻码（新老客分流）+自动打标签+多维度数据统计+备用员工容错。原生联系我都没有这些。

模块2：内部OA（审批/打卡）

场景："公司报销流程怎么配？"

话术：用企业微信审批。关键是选对审批方式：日常小额报销用或签（任一审批人同意即过，谁在岗谁批，快）；跨部门采购用会签（所有审批人都同意才过，缺一不可）；请假/合同用依次审批（直属领导→总监→HR，讲层级）。合同审批完要财务打款，用办理人节点——只需提交办理结果不需审批。

易被追问："办理人节点和审批节点有什么区别？"→ 办理人是执行不打审批，审批节点要同意才过。

模块3：管理和配置（通讯录/权限）

场景："子公司A的员工不应该看到子公司B的通讯录，怎么隔离？"

话术：用通讯录权限三档。第一档「隐藏部门/成员」把B彻底藏起来；第二档「限制查看外部门」让A只能看自己部门；第三档「限制查看所有人」最严格。三档都支持白名单例外。这是RBAC（基于角色的权限控制）的落地——权限跟着角色走不跟着人走。

模块4：活码&任务&营销工具

场景："连锁餐饮50家门店，希望客户扫码后按定位进最近的门店群，新客老客差异化处理。"

话术：用企微管家的区域群活码+时刻码组合。区域群活码按GPS定位自动分配到最近门店群（原生加入群聊做不到）；时刻码区分新老客——新客直接进群，老客跳转到福利页做复购。如果群满了，群活码还能自动建新群（顺序入群）或让客户选不同群（分类入群），还有备用员工承接防止流失。

必背7维差异：活码vs联系我=多人路由/时刻码新老客/分时段欢迎语/自动打标签/多维统计/自定义样式/备用员工。这7维是考试必考。

模块5：CRM

场景："画出你们的CRM从获客到成交的流程。"

话术：CRM3.0的核心是一条漏斗：线索→联系人→企业→商机→订单。线索是可能买的人（留过资）；联系人是找到具体对接人；企业是了解客户公司评估购买力；商机是有明确成交机会；订单是签约了。每一步都在筛选，越往后人越少价值越高。CRM管的是这条漏斗的转化率，不只是通讯录。

易被追问："CRM和通讯录什么区别？"→ CRM管销售过程（漏斗转化），通讯录只是其中"联系人"一环。

模块6：会话存档

场景："银行要做合规审计，需要留存所有员工和客户的聊天记录。"

话术：用会话存档。服务版覆盖内部+员工与外部客户沟通，银行买这个。注意三个硬限制：企微侧只存5天（企业必须5天内自己拉取存储，否则丢失）；一次拉取上限1000条；调用频率4000次/分钟。版本间不支持升级，买服务版后想升企业版要重买。

管家增强：企微管家在存档基础上做超时回复规则（X分钟未回复触发提醒）和敏感词规则（辱骂/飞单/竞品拦截告警），还能搭智能质检任务，AI全覆盖检查服务质量。

模块7：AI功能

场景："你们的AI能做什么？"

话术：企微管家AI聚焦两个企业场景：一是智能质检——AI自动检查所有客服聊天，发现超时未回复、敏感词、服务态度差，代替人工抽检，100%全覆盖；二是知识库+智能问答——把产品文档上传知识库，员工跟客户聊天时侧边栏根据客户问题自动推荐话术，新人不用翻手册也能专业回复。

易被追问："和ChatGPT有什么区别？"→ 不是通用聊天AI，是聚焦"检查服务质量"和"辅助员工回答"两个企业场景的专用AI。

模块8：素材&客户管理&数据统计

场景："怎么把客户的反馈自动打标签？"

话术：用客户收集表。创建一个收集表（比如"满意度调查"），客户填写后，内容自动同步到客户标签。这样后续可以按标签精准群发——比如给"不满意"的客户发优惠券挽回，给"高意向"的客户发新品推荐。

03 考官追问应对大全

考官一定追问。下面是最高频的追问+标准回答，背下来考场直接用。

Q1："这个功能企业微信原生不就有吗，为什么要买企微管家？"

A：原生给基础能力，管家做自动化、精细化、数据化增强。比如联系我原生就是一个固定二维码；管家活码加了多人路由、分时段分配、新老客分流、自动打标签、数据统计、备用员工容错——7个维度都是原生没有的。管家是增强插件，没有企业微信就没有管家。

Q2："活码和联系我到底什么区别？"

A：7个维度。①多人路由+分时段分配（联系我是固定码）②时刻码区分新老客（联系我无识别）③分时段个性化欢迎语（联系我是固定内容）④自动打标签+备注+渠道记录（联系我需手动）⑤多维度数据统计（联系我只有扫码总数）⑥自定义样式+短链（联系我固定）⑦备用员工容错（联系我无）。一句话：联系我是"加好友入口"，活码是"智能获客路由系统"。

Q3："群活码和加入群聊什么区别？"

A：加入群聊是企微原生（绑5群、满200自建、永不过期）；群活码是管家增强（顺序入群+分类入群+区域活码按定位分配+备用员工+自动打标签+时刻群码区分新老客）。核心差异：原生管"一个码对一群"，管家管"一个码对多群+智能分配+容错"。

Q4："会话存档能存多久？"

A：企微侧只存5天，企业必须5天内通过API拉取到自己服务器，否则丢失。这是"拉取窗口"不是"留存期"。一次拉取上限1000条，频率4000次/分钟。

Q5："朋友圈一天能发几条？"

A：每天每客户最多3条，企业和员工共用3次——不是各3条。企业发2条，员工只能发1条。

Q6："离职销售的客户怎么办？"

A：用离职继承。客户24小时后自动切换给接替员工，无感转接。每90天内客户/客户群最多被转接2次。这是企业微信相对个人微信最大的私域优势——客户归企业不归个人。

Q7："CRM和通讯录什么区别？"

A：CRM管销售漏斗（线索→联系人→企业→商机→订单），管的是转化过程；通讯录只是CRM里"联系人"这一环。CRM不是通讯录。

Q8："企微管家和企业微信是什么关系？"

A：企微管家是基于企业微信API开发的SaaS增强插件。企业微信是平台，企微管家是插件。没有企业微信就没有企微管家。管家不是替代品，是增强层。

04 7道实操考核题口述答案

实操考核的7道题，考官可能让你口述配置步骤。下面是每道题的口述标准答案。

实操1：配置"联系我"二维码 + "加入群聊"

联系我：管理后台→客户联系→联系我→选择员工或部门→设置欢迎语和备用承接→生成二维码投放。

加入群聊：管理后台→客户群→加入群聊→选择载体（二维码/立牌/小程序按钮）→绑定客户群（最多5个）→群满200人自动建新群→生成投放。

实操2：列出活码vs联系我的功能差异点

7个维度：多人路由+分时段 / 时刻码新老客 / 分时段个性化欢迎语 / 自动打标签+备注+渠道 / 多维度数据统计 / 自定义样式+短链 / 备用员工容错。每个维度联系我都没有。

实操3：列出群活码vs加入群聊的功能差异点

7个维度：顺序入群+分类入群 / 群满自动建新群（原生是满200自建但无分类）/ 备用员工承接 / 多维度数据统计 / 入群自动打标签+关联群发 / 时刻群码区分新老客 / 区域活码按定位分配。

实操4：配置群发任务（4种）

群发客户：客户联系→群发助手→新建群发→选客户→编辑内容（附件最多9个）→发送（每次最多2000客户，企业和员工共用每日1条）。

群发群：群发助手→新建群发→选客户群→编辑内容→发送。

群发朋友圈：客户朋友圈→发表→编辑内容→选择可见客户（每天每客户最多3条，企业员工共用3次）。

超级群发：以上三种的组合触达，一次操作多渠道同时触达（管家独有，原生没有）。

实操5：输出CRM3.0整体流程逻辑图

漏斗五层：线索（可能买的人）→联系人（找到具体对接人）→企业（了解客户公司）→商机（有明确成交机会）→订单（签约了）。每步都有筛选和转化率。

实操6：配置超时回复规则 + 敏感词规则

超时回复：企微管家→会话存档→质检规则→新建超时规则→设阈值（如30分钟未回复）→选适用员工→触发提醒/上报。

敏感词：企微管家→会话存档→敏感词规则→新建→添加词库（辱骂/飞单/竞品/违规承诺）→设置处理方式（拦截/告警）。

实操7：搭建智能质检任务 + 知识库智能问答

智能质检：企微管家→AI功能→智能质检→新建质检任务→选质检范围（员工/时间）→配置质检维度（超时/敏感词/态度）→启动自动检查。

知识库：企微管家→AI功能→知识库→上传文档（pdf/doc/docx/xlsx，单文件60M内）→员工聊天时侧边栏自动推荐话术。

05 答辩周旋技巧（"主持人的周旋"）

备考计划里言秋说"得会主持人的周旋"。这不是让你当主持人，是让你掌握答辩的主动权。

技巧1：考官问的没复习到→往你熟悉的引

考官问了一个你没准备的细节，不要慌，不要硬编。用这句话接：

"这个细节我回头确认一下。不过这个功能在实际运营中更多是配合XX一起用的，我重点给您讲下XX怎么用——"

然后顺势把话题引到你熟悉的功能上。考官要看的是你会不会做产品介绍，不是你背不背得下每个参数。

技巧2：考官追问差异→用"7维对比"一锤定音

任何"XX和YY有什么区别"的问题，都用结构化对比回答，不要零散说。模板：

"我从X个维度来对比。维度1……维度2……维度3……。一句话总结：XX是（定位），YY是（定位）。"

比如活码vs联系我：7维对比+一句话总结"联系我是加好友入口，活码是智能获客路由系统"。

技巧3：考官质疑"原生就够用"→亮出管家差异化

这是最高频的挑战。标准应对：

"原生确实提供了基础能力。但企业规模化运营时会遇到三个痛点：①人工操作多易出错②数据分散看不全③没有容错机制。企微管家就是解决这三个问题的——自动化（活码自动分配）、数据化（多维统计）、容错化（备用员工）。原生是'能用'，管家是'好用且管用'。"

技巧4：不确定的别说死

考官问"这个上限是多少"你不记得，宁可说"具体数值我确认下，大概在XX范围"，不要编一个错的数字。编错了比说"不确定"扣分多得多。

06 考前必背速记卡

以下内容进考场前必须背到脱口而出。

  • 活码vs联系我：7维（多人路由/时刻码/欢迎语/自动标签/统计/样式/备用）
  • 群活码vs加入群聊：7维（顺序分类入群/备用/统计/自动标签/时刻群码/区域活码）
  • CRM漏斗：线索→联系人→企业→商机→订单
  • 群发4种：群发客户/群/朋友圈/超级群发
  • 客户群上限：500人；全员群1万；部门群2千
  • 朋友圈：每天3条共用（不是各3条）
  • 群发客户：每次2000个，附件9个，每日1条共用
  • 离职继承：24h自动，90天2次上限
  • 会话存档：企微存5天，拉取1000条/次，4000次/分，不支持版本升级
  • 会话存档3版本：办公（内部）/服务（+外部）/企业（+VOIP）
  • 审批3种：或签（快）/会签（严）/依次（层级），办理人节点=执行不打审批
  • 企微加密：仅内部单聊+15人以下群，会话存档看不到
  • 折扣权限：标品6折/商业化85折
  • 企微管家定位：企微API的SaaS增强插件，没企微就没管家

一句话记住管家价值：原生给基础能力，管家做自动化、精细化、数据化增强——这三化就是答案。

一句话记住考试核心：你不是在答题，你是在做一场产品演示+客户答疑，考官想看的是你像客户成功一样把产品讲清楚。

08 考前一晚清单

考前一晚必做（按顺序）

  1. 背「06 考前必背速记卡」——所有硬数字和差异对比脱口而出
  2. 过一遍「02 8模块演示话术」——每个模块能说出客户场景+功能介绍+管家优势
  3. 过一遍「03 考官追问应对」8个Q&A——尤其Q1（原生vs管家）和Q2（活码vs联系我7维）
  4. 记住「05 答辩周旋技巧」4个技巧——尤其"没复习到就往熟悉的引"
  5. 做完「07 通关模拟关卡」8道题——全对才算过关
  6. 早睡。明天考的是表达不是熬夜。

最后一句：你不是在"考试"，你是在"做一场产品演示"。考官扮演客户，你像客户成功一样把产品讲清楚、把问题接住。自信、结构化、有场景。稳稳通过。
