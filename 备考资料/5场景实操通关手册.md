# 管培生考试·5场景实操通关手册

> 考试规则：5 个场景考官**现场随机抽 1 个**，2 个考官打分取平均。
> 策略：**每个场景都必须练到能盲操作**，不能押宝。
> 通用账号：用企微管家**体验账号后台**（管理端）操作。
> 通用入口：企微管家后台 → 左侧菜单。
>
> 本手册每个功能给出 **①操作路径 ②演示话术 ③管家优势（考官加分项）④易错点**。

---

## 📋 考前30秒必读·通用演示框架

不管抽到哪个场景，都用这个开场结构：

> "考官好，我演示的场景是【场景X】。我会按**获客→互动→转化→管理→AI赋能**的运营全流程，依次演示这几个功能，并在每个环节说明企微管家相比企业微信原生的差异化优势。"

**操作时三个铁律**：
1. **边做边讲**——不要闷头点，每点一个按钮说一句"我现在配置的是XX，它的作用是YY"
2. **每个功能都要讲差异化**——考官要听的是"管家比原生强在哪"，不是"我会点按钮"
3. **配置完要展示效果**——比如配完活码要说明"客户扫码后会XX"，形成闭环

## 🎯 场景一：区域活码（关联员工活码）+ 客户群发 + 轨迹素材 + 客户画像 + AI智能回复

**运营主线**：线下门店按地理位置引流→加好友→群发触达→素材追踪意向→画像分析→AI辅助回复

### 1️⃣ 区域活码（关联员工活码）

**操作路径**：
1. 后台 → **门店管理** → 新建门店（填门店名、地址、经纬度）→ 保存
2. 后台 → **营销获客 → 区域活码** → 新建区域活码
3. 配置：选择关联类型 = **员工活码**；选择门店；关联使用员工；设置每日添加上限
4. 生成活码 → 下载二维码/短链

**演示话术**：
> "区域活码基于**LBS地理位置**，客户扫码时系统自动识别他的位置，分配到最近的门店员工。我这里先建门店，再建区域活码并关联员工活码。注意区域活码**必须先创建门店**，这是前置条件。"

**管家优势（加分项）**：
- 原生"联系我"是固定二维码，**无地理位置识别、无添加上限、无自动分配**
- 管家区域活码支持**LBS定位分流**、**多人活码设添加上限**、**分时段欢迎语**、**新老客识别（时刻码）**

**易错点**：
- ❌ 忘记先建门店就建区域活码（会失败）
- ❌ 混淆"区域活码关联员工活码"（场景一）vs"关联群活码"（场景二）

### 2️⃣ 客户群发

**操作路径**：
1. 后台 → **营销获客 → 客户群发**（或"标准化营销"）→ 新建群发任务
2. 选择接收对象：按标签/按员工/按客户列表筛选
3. 编辑内容（文本+图片+链接，可从素材库选）
4. 设置发送时间（立即/定时）→ 保存下发
5. 员工在企微侧边栏收到任务 → 确认发送

**演示话术**：
> "客户群发是**员工代发**模式：管理员后台建任务，员工在企微侧边栏确认后发出。这样能避免骚扰、保护客户体验。注意企微原生群发规则限制：**每天1条、每周7条、每月按当月天数条**。"

**管家优势**：
- 原生群发助手每次都要手动选人，管家支持**按标签批量筛选**、**标准化营销任务自动触发**

**易错点**：
- ❌ 把"客户群发"和"群发客户群""超级群发"搞混（见下方对比）

### 3️⃣ 轨迹素材

**操作路径**：
1. 后台 → **内容中心 / 素材管理 → 轨迹素材** → 上传素材（文件≤20个、视频≤100M）
2. 员工发送给客户 → 客户浏览 → 员工企微侧边栏**收到浏览提醒**

**演示话术**：
> "轨迹素材是**带行为追踪的素材**。客户浏览后，员工会实时收到提醒，能在客户意向最高点及时跟进。注意：**文件和视频**浏览才触发提醒，普通海报/文本不触发。"

**管家优势**：
- 这是**管家独有能力**，原生企微无素材行为追踪

**易错点**：
- ❌ 以为所有素材都触发提醒（只有文件/视频）

### 4️⃣ 客户画像

**操作路径**：
1. 后台 → **客户管理 → 客户列表** → 点击某客户 → 查看画像
2. 展示画像维度：基础信息、标签、轨迹、消费、跟进记录

**演示话术**：
> "客户画像整合了**行为数据、消费数据、第三方数据**三大维度。默认看3个月轨迹，最长可查6个月。我还能看到客户当前处于哪个阶段（新客户→开口→意向→有需求→高价值→商务谈判）。"

**管家优势**：
- 原生只有基础信息，管家画像**多维度+轨迹+阶段管理**，标签可同步到企微

**易错点**：
- ❌ 把客户阶段顺序记错（正确：新→开口→意向→有需求→高价值→商务谈判）

### 5️⃣ AI智能回复

**操作路径**：
1. 后台 → **AI功能 → 智能问答** → 配置知识库（上传产品文档/FAQ）
2. 员工在企微**聊天侧边栏** → 点击智能问答 → 输入客户问题 → AI生成推荐话术 → 一键发送

**演示话术**：
> "智能问答基于**企业知识库**生成专业回复，在聊天侧边栏直接调用。员工不用背话术，AI实时推荐，还能用多模型（DeepSeek等）对比输出。智能问答入口有三处：**侧边栏、员工知识工作台、内部群机器人**。"

**管家优势**：
- 完全管家独有，原生无AI能力

**易错点**：
- ❌ 忘记智能问答的三个入口（侧边栏/知识工作台/群机器人）

## 🎯 场景二：区域活码（关联群活码）+ 群裂变 + 群群发 + 超级群发 + AI话题洞察

**运营主线**：线下LBS引流入群→裂变扩群→群内批量触达→管理员强制触达→AI分析群聊话题

### 1️⃣ 区域活码（关联群活码）

**操作路径**：
1. 先建门店（同场景一）
2. 后台 → **区域活码** → 关联类型 = **群活码**
3. 选择客户群活码 → 群满后自动建新群 → 保存

**演示话术**：
> "区别于场景一关联员工，这里关联群活码。客户扫码按LBS进最近的门店群。群活码支持**顺序入群**（群满自动建新群）和**分类入群**（多场景选择），原生'加入群聊'一个码只对应一个群、满员即失效。"

**管家优势**：
- 原生"加入群聊"**无自动建群、无备用员工、无入群打标签**；管家群活码全有，还支持**时刻群码识别新老客**

**易错点**：
- ❌ 区域活码创建前**必须先建门店**

### 2️⃣ 群裂变

**操作路径**：
1. 后台 → **营销获客 → 裂变工具 → 群裂变** → 新建活动
2. 配置：邀请N人得奖励、活动时间、防刷设置 → 发布
3. 客户在群内看到活动 → 分享海报 → 好友扫码进群 → 自动计任务→ 达标发奖

**演示话术**：
> "群裂变利用客户社交关系链扩群。管家三大裂变：**任务裂变、群裂变、红包裂变**（没有直播裂变）。客户完成邀请任务后自动发奖。"

**管家优势**：
- 原生无任何裂变能力

**易错点**：
- ❌ 以为有"直播裂变"（只有任务/群/红包三种）

### 3️⃣ 群群发（群发客户群）

**操作路径**：
1. 后台 → **营销获客 → 标准化营销 / 群发** → 新建 → 类型选**群发客户群**
2. 选择目标群 → 编辑内容 → 下发 → 群主在企微确认发送

**演示话术**：
> "群群发针对**客户群**，由群主确认后发出。标准化营销任务的触发类型有三种：**群发客户、群发客户群、跟进客户**。"

### 4️⃣ 超级群发

**操作路径**：
1. 后台 → **营销获客 → 超级群发** → 新建任务
2. 选择群 + **可指定群成员**执行 → 编辑内容 → 下发

**演示话术**：
> "超级群发是管家的**增强版群发**，关键能力是**管理员可以指定具体群成员来执行任务**，做到精准到人。普通群发做不到指定成员。"

**管家优势**：
- **指定群成员执行**是超级群发核心差异点

**易错点**：
- ❌ 说不清超级群发和普通群发的区别（答：超级群发可**指定群成员**执行）

### 5️⃣ AI话题洞察

**操作路径**：
1. 后台 → **AI功能 → 话题洞察** → 开启/查看分析
2. 展示：话题趋势、热词、客户情感（正/负/中）

**演示话术**：
> "话题洞察**T+1自动分析**每天的聊天内容，生成话题分布、热词排行和情感趋势。比如能发现客户最近都在讨论价格（负向情感多），运营就能针对性优化。它**自动分析话题与情感**，输出词云和趋势报告。"

**管家优势**：
- 原生无此能力。话题洞察分析维度：产品价格、库存供货、产品咨询、客户评价、问题投诉等

**易错点**：
- ❌ 把话题洞察和智能质检搞混（话题洞察看**话题和情感**，质检看**员工服务规范**）

## 🎯 场景三：识客码 + 微信客服 + 标准化营销 + 好友裂变 + 群发朋友圈 + AI朋友圈广场

**运营主线**：渠道识别客户来源→临时咨询→自动化营销触达→裂变拉新→朋友圈触达→AI批量生产朋友圈内容

### 1️⃣ 识客码

**操作路径**：
1. 后台 → **营销获客 → 识客码** → 新建
2. 配置：渠道标签、欢迎语、跳转页面 → 生成二维码

**演示话术**：
> "识客码用于**渠道来源追踪**，不同渠道生成不同码，客户扫码自动打渠道标签，方便后续按渠道分析ROI。区别于区域活码（按地理位置），识客码按**渠道**分流。"

### 2️⃣ 微信客服

**操作路径**：
1. 后台/企微管理端 → **微信客服** → 新建客服账号 → 配置欢迎语、接待规则
2. 客户在微信点击客服入口 → 临时咨询（无需加好友）

**演示话术**：
> "微信客服提供**临时咨询**，客户不用加好友就能沟通，降低服务门槛。它是企微拉新的四大产品能力之一（广告管家、活码工具、裂变工具、微信客服）。"

### 3️⃣ 标准化营销

**操作路径**：
1. 后台 → **标准化营销** → 新建营销任务
2. 选触发类型（群发客户/群发客户群/跟进客户）→ 配置触发条件 → 下发

**演示话术**：
> "标准化营销是**自动化营销引擎**，按条件自动触发任务，减少人工重复操作。三大触发类型：群发客户、群发客户群、跟进客户。"

### 4️⃣ 好友裂变

**操作路径**：
1. 后台 → **裂变工具 → 好友裂变/任务裂变** → 新建
2. 配置邀请任务、奖励、规则 → 发布

**演示话术**：
> "好友裂变通过**老带新**扩好友，客户完成任务自动发奖。"

### 5️⃣ 群发朋友圈

**操作路径**：
1. 后台 → **营销获客 → 朋友圈 / 群发朋友圈** → 新建
2. 编辑内容 → 选择可见客户 → 下发 → 员工确认发布到客户朋友圈

**演示话术**：
> "群发朋友圈让管理员统一下发朋友圈内容，员工确认后发布，保证内容规范统一。"

### 6️⃣ AI朋友圈广场

**操作路径**：
1. 后台/侧边栏 → **AI功能 → 朋友圈广场**
2. AI生成朋友圈文案 → 选择 → 发布

**演示话术**：
> "AI朋友圈广场帮员工**批量生成朋友圈内容**，解决'不知道发什么'的问题，提升朋友圈活跃度。"

**管家优势**：
- 全链路AI赋能，原生无

## 🎯 场景四：员工活码短链 + 收集表 + 客户画像 + 标准化营销 + 敏感词预警 + AI分析智能体

**运营主线**：短链全渠道获客→表单收集线索→画像补全→自动化跟进→风控防飞单→AI分析决策

### 1️⃣ 员工活码短链

**操作路径**：
1. 后台 → **营销获客 → 员工活码** → 新建多人活码
2. 配置员工、欢迎语 → **生成短链**（用于短信/邮件等非微信渠道）

**演示话术**：
> "员工活码除二维码外还能**生成短链**，用于短信、邮件等非微信渠道获客，并统计各渠道引流数据。原生'联系我'只能微信扫码，无短链能力。"

**易错点**：
- ❌ 只有**多人活码**才能设添加上限，单人不行

### 2️⃣ 收集表

**操作路径**：
1. 后台 → **客户管理 / 内容中心 → 收集表** → 新建
2. 配置字段（姓名/手机/需求等）→ **可同步到标签** → 发布 → 客户填写 → 数据回收

**演示话术**：
> "收集表用于**线索收集**，关键字段可**自动同步为客户标签**，省去手动打标。这是实操考核原题：'创建收集表，并将填写内容同步到标签'。"

### 3️⃣ 客户画像

（同场景一第4步：多维度画像 + 阶段管理，展示行为/消费/第三方数据）

### 4️⃣ 标准化营销

（同场景三第3步：按条件自动触发群发/跟进任务）

### 5️⃣ 敏感词预警

**操作路径**：
1. 后台 → **会话存档 → 敏感行为/敏感词管理** → 新建规则
2. 配置敏感词列表、触发动作（提醒/拦截）→ 保存

**演示话术**：
> "敏感词预警基于**会话存档**，员工一旦在聊天中发送敏感词（如'飞单''加微信'），系统自动预警拦截，防止飞单和违规。会话存档有**2项拦截方式**。"

**管家优势**：
- 原生license只能看接口不能直接查内容，管家提供**敏感词、超时回复、离职检索**等管理能力

**易错点**：
- ❌ 敏感词预警**依赖会话存档**，必须先开通会话存档

### 6️⃣ AI分析智能体

**操作路径**：
1. 后台 → **AI功能 → 多模型AI助手 / Agent应用**
2. 上传文件或输入问题 → AI分析输出（可选两个模型对比）

**演示话术**：
> "AI分析智能体支持**多模型对比**，可同时选两个模型输出答案对比，支持文件/截图/多模态输入。管家的Agent应用包括：话题洞察、客户洞察、智能跟进、智能转化分析等。"

## 🎯 场景五：超级活码 + 跟进任务 + 抽奖 + 标准化营销 + 超时回复 + AI客户洞察

**运营主线**：活码全渠道获客→任务驱动跟进→抽奖活跃→自动化触达→服务时效监控→AI深度洞察客户

### 1️⃣ 超级活码

**操作路径**：
1. 后台 → **营销获客 → 超级活码**（聚合多种活码能力）→ 新建
2. 配置引流规则、关联活码类型 → 生成

**演示话术**：
> "超级活码是**活码的聚合升级**，整合员工活码、群活码、区域活码等能力，一个入口满足多种引流场景。"

### 2️⃣ 跟进任务

**操作路径**：
1. 后台 → **CRM / 标准化营销 → 跟进任务** → 新建
2. 指定员工、客户、跟进时间、跟进内容 → 下发
3. 员工企微收到任务提醒 → 执行跟进 → 记录结果

**演示话术**：
> "跟进任务把销售动作**任务化、可追踪**。跟进功能覆盖**企业、联系人、客户、客户群、商机**全生命周期，协作团队成员都能创建。"

**易错点**：
- ❌ 以为跟进只针对客户（实际覆盖5类对象）

### 3️⃣ 抽奖

**操作路径**：
1. 后台 → **营销获客 → 抽奖活动** → 新建
2. 选类型（**通用抽奖 / 获客抽奖**）→ 配置奖品、概率、规则 → 可设**人工审核**→ 发布

**演示话术**：
> "抽奖有**通用抽奖和获客抽奖**两种类型。晒单/抽奖支持**人工审核**，防作弊防违规。"

**易错点**：
- ❌ 记不住抽奖类型（答：通用/获客两种）

### 4️⃣ 标准化营销

（同场景三第3步）

### 5️⃣ 超时回复

**操作路径**：
1. 后台 → **会话存档 → 超时回复规则** → 新建
2. 配置：触发时间（**最低5分钟**）、提醒对象、提醒方式 → 保存

**演示话术**：
> "超时回复监控服务时效，员工超过设定时间未回复客户就触发提醒。**最低触发时间是5分钟**。这保证客户咨询不被遗漏，提升响应效率。"

**易错点**：
- ❌ 超时回复最低触发时间记错（答：**5分钟**）

### 6️⃣ AI客户洞察

**操作路径**：
1. 后台 → **AI功能 → 客户洞察** → 开启
2. 查看某客户：AI自动提取的意向产品、需求痛点、购买顾虑、跟进建议

**演示话术**：
> "客户洞察**实时分析**客户会话，自动提取意向产品、购买异议点、需求痛点，并生成跟进建议。四大亮点：**信息不遗漏、了解客户更轻松、边聊边管理、提升转化效率**——把对客户的'懂'转化为销售的'准'。"

**管家优势**：
- 这是**答辩高光项**，体现AI直接赋能销售转化

**易错点**：
- ❌ 客户洞察和话题洞察搞混（客户洞察看**单个客户**，话题洞察看**群聊趋势**）
