01 考试总览（考核公告）

🔴 2026管培生考核公告

项目内容
考试时间2026年6月30日
考试地点无锡办公室
考试范围公司基础产品知识（培训课程 + 定期答疑腾讯会议录屏）
考核构成两部分：①笔试（选择/判断/简答，100分）②实操演示通关（100分）
计分规则笔试、通关各占 50%，最终得分 = 单项得分 × 权重之和
及格线≥ 70 分 视为通过

两段式考核流程

第一部分笔试（会议室集中、线上考试系统、1小时内）：选择题、判断题由系统自动判分；简答题由考官根据答案关键词评分。笔试完成后，按指引到另一会议室参与第二部分实操演示通关。

通关形式：考官从 5 个实操场景中现场随机抽 1 个，你进行产品功能演示，由 2 个考官打分取平均分（满分100）。

备考优先级（按得分占比）

环节占比怎么拿分对应章节
① 笔试50%选择/判断题背硬数字+判功能归属+记7维差异+识别陷阱；简答题按关键词组织答案02 笔试 + 06 题库 + 07 速记卡
② 实操通关50%5个场景每个都要练到能盲操作，演示时突出管家差异化优势03 通关规则 + 04 五场景手册 + 05 平台实操

核心心法：笔试考"记得准"，通关考"演得像客户成功"。两块各占一半，都不能瘸腿——通关没练熟，笔试满分也救不回来；反之亦然。

02 笔试怎么考

笔试在会议室集中进行，用线上考试系统，限时 1 小时内。题型有三类，评分方式不同：

题型评分方式怎么准备
选择题系统自动判分背硬数字、判原生vs管家归属、记7维差异。见 06 笔试题库。
判断题系统自动判分专治错误说法：认准"动态二维码/永久保存/CRM=通讯录/时刻码按时间/管家替代企微"5个陷阱词，全是❌。
简答题考官按答案关键词评分答题要踩中关键词，分点作答、结构化输出。见下方简答策略。

简答题得分策略（关键词评分制）

简答题不是主观发挥，是把关键词"砸"到卷面上。考官拿着关键词清单打分，缺一个关键词扣一分。

  • 分点作答：用"①②③"或"第一/第二"，让考官一眼数到关键词数量。
  • 先定位再展开：先说"这是在XX后台（原生/管家）配置"，再说路径，最后讲差异化——和通关演示一个套路。
  • 必砸关键词清单：活码7维（多人路由/时刻码/欢迎语/自动标签/统计/样式/备用）、CRM漏斗（线索→联系人→企业→商机→订单）、会话存档5天拉取窗口、朋友圈3条共用、离职继承24h-90天2次。
  • 不确定的用范围，别编："具体数值我确认，大概在XX范围"比编一个错数字扣分少。

03 通关怎么考（5场景现场抽1个）

通关规则（官方）

  • 抽取方式：考官从下面 5 个实操场景中现场随机抽 1 个。
  • 演示要求：你针对抽中的场景，进行相关功能的产品实操演示。
  • 打分方式：由 2 个考官打分，得分为两人平均分（满分100）。
  • 占最终成绩 50%，与笔试各占一半。

策略铁律：每个场景都必须练到能盲操作，不能押宝。因为你不知道会抽到哪个。

考前30秒通用开场框架（不管抽到哪个场景都用）：

"考官好，我演示的场景是【场景X】。我会按获客 → 互动 → 转化 → 管理 → AI赋能的运营全流程，依次演示这几个功能，并在每个环节说明企微管家相比企业微信原生的差异化优势。"

操作三铁律：①边做边讲（每点一个按钮说一句"我现在配置的是XX，作用是YY"）②每个功能点出管家差异化（考官考的是你"懂不懂产品价值"，不是"会不会点按钮"）③配置完要展示效果（配完活码说明"客户扫码后会XX"，形成闭环）。

⚠️ 考场 ≠ 客户：第②条是给考官的策略——考场就那么点时间，要快速证明你懂产品，所以逢功能主动点差异。到了真实客户场景反过来：客户没主动问就别堆对比，容易把人绕晕；客户要的是"解决我的问题"，按需对比即可。

🎯 场景一：区域活码（关联员工活码）+ 客户群发 + 轨迹素材 + 客户画像 + AI智能回复

运营主线：线下门店按地理位置引流 → 加好友 → 群发触达 → 素材追踪意向 → 画像分析 → AI辅助回复

1️⃣ 区域活码（关联员工活码）

platform → 我的企业 → 门店管理 → 新建门店（填门店名、地址、经纬度）→ 保存
platform → 客户营销 → 获客拉新 → 区域活码 → 新建 → 关联类型选 员工活码 → 选择门店 → 关联使用员工 → 设每日添加上限 → 生成二维码/短链

"区域活码基于 LBS 地理位置，客户扫码时系统自动识别位置，分配到最近的门店员工。我这里先建门店，再建区域活码并关联员工活码。注意区域活码必须先创建门店，这是前置条件。"

管家优势（加分项）：原生"联系我"是固定二维码，无地理位置识别、无添加上限、无自动分配；管家区域活码支持 LBS定位分流、多人活码设添加上限、分时段欢迎语、新老客识别（时刻码）。

易错点：❌忘记先建门店就建区域活码（会失败）；❌混淆"关联员工活码"（场景一）vs"关联群活码"（场景二）。

2️⃣ 客户群发

platform → 客户营销 → 营销转化 → 群发工具 → 群发客户 → 新建任务 → 选接收对象（按标签/按员工/按客户列表）→ 编辑内容（文本+图片+链接，可从素材库选）→ 设发送时间（立即/定时）→ 下发 → 员工企微侧边栏收到任务 → 确认发送

"客户群发是员工代发模式：管理员后台建任务，员工在企微侧边栏确认后发出，避免骚扰、保护客户体验。注意企微原生群发规则限制：每天1条、每周7条、每月按当月天数条。"

管家优势：原生群发助手每次手动选人，管家支持按标签批量筛选、标准化营销任务自动触发。

易错点：❌把"客户群发"和"群发客户群""超级群发"搞混（见 群发对比表）。

3️⃣ 轨迹素材

platform → 内容中心 → 通用素材 → 轨迹素材 → 上传素材（文件≤20个、视频≤100M）→ 员工发送给客户 → 客户浏览 → 员工企微侧边栏收到浏览提醒

"轨迹素材是带行为追踪的素材。客户浏览后员工会实时收到提醒，能在客户意向最高点及时跟进。注意：文件和视频浏览才触发提醒，普通海报/文本不触发。"

管家优势：这是管家独有能力，原生企微无素材行为追踪。

易错点：❌以为所有素材都触发提醒（只有文件/视频）。

4️⃣ 客户画像

platform → 客户管理 → 客户列表 → 点击某客户 → 查看画像（基础信息 / 标签 / 轨迹 / 消费 / 跟进记录）

"客户画像整合了行为数据、消费数据、第三方数据三大维度。默认看3个月轨迹，最长可查6个月。我还能看到客户当前处于哪个阶段。"

管家优势：原生只有基础信息，管家画像多维度+轨迹+阶段管理，标签可同步到企微。客户阶段顺序：新客户→开口→意向→有需求→高价值→商务谈判。

易错点：❌把客户阶段顺序记错。

5️⃣ AI智能回复

platform → 客户营销 → AI智能体 → AI知识库 → 配置知识库（上传产品文档/FAQ）→ 员工企微聊天侧边栏 → 智能问答 → 输入客户问题 → AI生成推荐话术 → 一键发送

"智能问答基于企业知识库生成专业回复，在聊天侧边栏直接调用。员工不用背话术，AI实时推荐，还能用多模型（DeepSeek等）对比输出。智能问答入口有三处：侧边栏、员工知识工作台、内部群机器人。"

管家优势：完全管家独有，原生无AI能力。

易错点：❌忘记智能问答的三个入口。

🎯 场景二：区域活码（关联群活码）+ 群裂变 + 群群发 + 超级群发 + AI话题洞察

运营主线：线下LBS引流入群 → 裂变扩群 → 群内批量触达 → 管理员强制触达 → AI分析群聊话题

1️⃣ 区域活码（关联群活码）

先建门店（同场景一）→ platform → 客户营销 → 获客拉新 → 区域活码 → 关联类型选 群活码 → 选择客户群活码 → 群满后自动建新群 → 保存

"区别于场景一关联员工，这里关联群活码。客户扫码按LBS进最近的门店群。群活码支持顺序入群（群满自动建新群）和分类入群（多场景选择），原生'加入群聊'一个码只对应一个群、满员即失效。"

管家优势：原生"加入群聊"无自动建群、无备用员工、无入群打标签；管家群活码全有，还支持时刻群码识别新老客。原生一个活码绑5个群，管家关联200个。

易错点：❌区域活码创建前必须先建门店。

2️⃣ 群裂变

platform → 客户营销 → 获客拉新 → 社交裂变 → 群裂变 → 新建活动 → 配置（邀请N人得奖励、活动时间、防刷）→ 发布 → 客户群内分享海报 → 好友扫码进群 → 自动计任务 → 达标发奖

"群裂变利用客户社交关系链扩群。管家三大裂变：任务裂变、群裂变、红包裂变（没有直播裂变）。客户完成邀请任务后自动发奖。"

管家优势：原生无任何裂变能力。

易错点：❌以为有"直播裂变"（只有任务/群/红包三种）。

3️⃣ 群群发（群发客户群）

platform → 客户营销 → 营销转化 → 群发工具 → 群发客户群 → 选目标群 → 编辑内容 → 下发 → 群主在企微确认发送

"群群发针对客户群，由群主确认后发出。标准化营销任务的触发类型有三种：群发客户、群发客户群、跟进客户。"

4️⃣ 超级群发

platform → 客户营销 → 营销转化 → 群发工具 → 超级群发 → 新建任务 → 选群 + 可指定群成员执行 → 编辑内容 → 下发

"超级群发是管家的增强版群发，关键能力是管理员可以指定具体群成员来执行任务，做到精准到人。普通群发做不到指定成员。"

管家优势：指定群成员执行是超级群发核心差异点，管家独有、原生没有。

易错点：❌说不清超级群发和普通群发的区别（答：超级群发可指定群成员执行）。

5️⃣ AI话题洞察

platform → 会话存档 → 会话智能 → 话题洞察 → 开启/查看分析（话题趋势、热词、客户情感正/负/中）

"话题洞察T+1自动分析每天的聊天内容，生成话题分布、热词排行和情感趋势。比如发现客户最近都在讨论价格（负向情感多），运营就能针对性优化。它自动分析话题与情感，输出词云和趋势报告。"

管家优势：原生无此能力。分析维度：产品价格、库存供货、产品咨询、客户评价、问题投诉等。

易错点：❌把话题洞察和智能质检搞混（话题洞察看话题和情感，质检看员工服务规范）。

🎯 场景三：识客码 + 微信客服 + 标准化营销 + 好友裂变 + 群发朋友圈 + AI朋友圈广场

运营主线：渠道识别客户来源 → 临时咨询 → 自动化营销触达 → 裂变拉新 → 朋友圈触达 → AI批量生产朋友圈内容

2️⃣ 微信客服

"微信客服提供临时咨询，客户不用加好友就能沟通，降低服务门槛。它是企微拉新的四大产品能力之一（广告管家、活码工具、裂变工具、微信客服）。不需加好友、不依赖会话存档。"

3️⃣ 标准化营销（SOP）

platform → 客户营销 → 营销转化 → 营销任务 → 标准化营销 → 新建 → 选触发类型（群发客户/群发客户群/跟进客户）→ 配置触发条件 → 下发

"标准化营销是自动化营销引擎，按条件自动触发任务，减少人工重复操作。三大触发类型：群发客户、群发客户群、跟进客户。这是管家独有，原生不支持。"

4️⃣ 好友裂变

platform → 客户营销 → 获客拉新 → 社交裂变 → 任务裂变 → 新建 → 配置（邀请任务、奖励、规则）→ 发布

"好友裂变通过老带新扩好友，客户完成任务自动发奖。"

5️⃣ 群发朋友圈

platform → 客户营销 → 营销转化 → 群发工具 → 群发朋友圈 → 编辑内容 → 选可见客户 → 下发 → 员工确认发布到客户朋友圈

"群发朋友圈让管理员统一下发朋友圈内容，员工确认后发布，保证内容规范统一。每天每客户最多3条，企业和员工共用3次。"

6️⃣ AI朋友圈广场

platform / 侧边栏 → AI功能 → 朋友圈广场 → AI生成朋友圈文案 → 选择 → 发布

"AI朋友圈广场帮员工批量生成朋友圈内容，解决'不知道发什么'的问题，提升朋友圈活跃度。"

技巧1：考官问的没复习到 → 往你熟悉的引

考官问了一个你没准备的细节，不要慌，不要硬编：

"这个细节我回头确认一下。不过这个功能在实际运营中更多是配合XX一起用的，我重点给您讲下XX怎么用——"

然后顺势引到你熟悉的功能上。考官要看的是你会不会做产品介绍，不是你背不背得下每个参数。

技巧2：考官追问差异 → 用"7维对比"一锤定音

任何"XX和YY有什么区别"的问题，都用结构化对比回答：

"我从X个维度来对比。维度1……维度2……维度3……。一句话总结：XX是（定位），YY是（定位）。"

比如活码vs联系我：7维对比 + 一句话总结"联系我是加好友入口，活码是智能获客路由系统"。

技巧3：考官质疑"原生就够用" → 亮出管家差异化

这是最高频的挑战，标准应对：

"原生确实提供了基础能力。但企业规模化运营时会遇到三个痛点：①人工操作多易出错 ②数据分散看不全 ③没有容错机制。企微管家就是解决这三个问题的——自动化（活码自动分配）、数据化（多维统计）、容错化（备用员工）。原生是'能用'，管家是'好用且管用'。"

技巧4：不确定的别说死

考官问"这个上限是多少"你不记得，宁可说"具体数值我确认下，大概在XX范围"，不要编一个错的数字。编错了比说"不确定"扣分多得多。

09 结果应用与补考机制

成绩如何应用（官方规则）

得分结果
≥ 70 分视为通过第一阶段考核，继续试用期 / 实习生可转正
＜ 70 分可申请第二次考核。若第二次仍 ＜ 70 分，结合业务部门意见综合评估：
　评估不合格按不符合试用期录用条件解除劳动关系；或按不符合实习生转正条件处理，所约定 offer 失效、撤销

备考启示

  • 两块都不能瘸腿：笔试和通关各占 50%，偏科任何一边都可能拉低总分踩到 70 分线以下。
  • 有补考机会但有代价：第一次没过可以申请第二次，但第二次再不过会进入"综合评估"，存在 offer 失效风险。目标是一次过。
  • 得分=单项×权重：不要只盯着某一块满分，要算加权后的总分。比如笔试 80 × 50% + 通关 60 × 50% = 70（刚好及格，危险）。

稳过策略：把目标定在 笔试 ≥ 85、通关 ≥ 80，加权后 82.5，留出安全余量。笔试靠 06 题库+07 速记卡刷到全对，通关靠 04 五场景手册每个场景都练熟。
